│                                                                                                                      │
│ 背景                                                                                                                 │
│                                                                                                                      │
│ 为现有的Flora-evaluation测试框架提供四个独立的基础服务模块，用于丰富测试数据和提供支持服务。                         │
│                                                                                                                      │
│ 模块划分                                                                                                             │
│                                                                                                                      │
│ 1. api_normalization/ - API语义归一化服务                                                                            │
│                                                                                                                      │
│ 从Swagger文档生成标准化的API能力模型，包括：                                                                         │
│ - SwaggerParser: 解析Swagger/OpenAPI文档                                                                             │
│ - SemanticClusterer: 基于向量相似度的API聚类                                                                         │
│ - CapabilityExtractor: 提取能力模型                                                                                  │
│ - NormalizationService: 对外服务接口                                                                                 │
│                                                                                                                      │
│ 2. api_topology/ - API拓扑图服务                                                                                     │
│                                                                                                                      │
│ 构建和查询API依赖关系图，包括：                                                                                      │
│ - GraphBuilder: 构建依赖图并存储到Neo4j                                                                              │
│ - PathFinder: 路径查找器                                                                                             │
│ - TopologyService: 对外服务接口                                                                                      │
│                                                                                                                      │
│ 3. scenario_generation/ - 测试场景生成服务                                                                           │
│                                                                                                                      │
│ 基于API路径逆向生成业务测试问题，包括：                                                                              │
│ - ScenarioGenerator: 使用LLM生成场景                                                                                 │
│ - ScenarioValidator: 验证场景合理性                                                                                  │
│ - ScenarioGenerationService: 对外服务接口                                                                            │
│                                                                                                                      │
│ 4. stateful_mock/ - 有状态Mock服务                                                                                   │
│                                                                                                                      │
│ 提供带状态管理和混沌注入的Mock API环境，包括：                                                                       │
│ - MockServer: FastAPI动态服务器                                                                                      │
│ - StateManager: SQLite状态管理                                                                                       │
│ - ChaosEngine: 混沌注入引擎                                                                                          │
│ - MockService: 服务管理接口                                                                                          │
│                                                                                                                      │
│ 实现优先级                                                                                                           │
│                                                                                                                      │
│ Phase 1（基础功能）                                                                                                  │
│                                                                                                                      │
│ 1. api_normalization: Swagger解析 + 基础聚类                                                                         │
│ 2. api_topology: 图构建 + 路径查找                                                                                   │
│ 3. stateful_mock: 基础Mock服务器                                                                                     │
│                                                                                                                      │
│ Phase 2（增强功能）                                                                                                  │
│                                                                                                                      │
│ 4. scenario_generation: 场景生成                                                                                     │
│ 5. stateful_mock: 混沌注入                                                                                           │
│ 6. api_normalization: LLM优化                                                                                        │
│                                                                                                                      │
│ 集成方式                                                                                                             │
│                                                                                                                      │
│ 这些模块作为独立服务，可被测试框架调用：                                                                             │
│                                                                                                                      │
│ # 在 coop_eval_actual/ 中使用                                                                                        │
│ from api_normalization import NormalizationService                                                                   │
│ from api_topology import TopologyService                                                                             │
│ from scenario_generation import ScenarioGenerationService                                                            │
│ from stateful_mock import MockService                                                                                │
│                                                                                                                      │
│ # 生成测试数据                                                                                                       │
│ capabilities = NormalizationService().normalize_swagger("swagger.json")                                              │
│ paths = TopologyService().find_paths("login", "delete_order")                                                        │
│ scenarios = ScenarioGenerationService().generate_scenarios(paths[0])                                                 │
│                                                                                                                      │
│ # 启动Mock服务                                                                                                       │
│ MockService().start_server(capabilities["capabilities"])                                                             │

│                                                                                                                      │

│ - 集成测试：端到端测试完整流程                                                                                       │
│ - 与现有测试框架集成验证                                                                                             │
